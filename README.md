# Skin Insight

**AI-powered skin analysis and product recommendation web application.**

Skin Insight uses a Convolutional Neural Network (CNN) to detect skin conditions from a facial photo and recommends skincare products with prices in Pakistani Rupees (Rs).

## Supported Skin Conditions

| Condition     | Description                              |
|---------------|------------------------------------------|
| Acne          | Pimples, cysts, inflamed skin            |
| Blackheads    | Clogged pores with dark appearance       |
| Dark Spots    | Hyperpigmentation, uneven skin tone      |
| Normal        | Healthy, well-balanced skin              |
| Pores         | Enlarged / visible pores                 |
| Wrinkles      | Fine lines and signs of ageing           |

## Quick Start

### 1. Prerequisites

- Python 3.10+
- A webcam (or image files for upload)

### 2. Create and Activate a Virtual Environment

```bash
python -m venv venv

# macOS / Linux
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Initialise the Database

```bash
python backend/create_db.py
```

This creates `backend/database.db` and seeds it with sample products from `datasets/products.csv`.

### 5. Prepare Your Dataset

Place training images into sub-folders named after each skin class:

```
datasets/
├── train/
│   ├── Acne/          ← JPEG/PNG images of acne skin
│   ├── Blackheads/
│   ├── Dark Spots/
│   ├── Normal/
│   ├── Pores/
│   └── Wrinkles/
└── test/
    ├── Acne/
    └── ...            ← same structure as train/
```

A good public dataset source is [Kaggle – Skin Disease datasets](https://www.kaggle.com/search?q=skin+disease+classification).

### 6. Train the Model

```bash
python scripts/train_model.py
```

The trained model is saved to `backend/models/skin_model.h5`.
A training history chart is saved to `backend/models/training_history.png`.

### 7. Start the Backend Server

```bash
python backend/app.py
```

The Flask server runs on `http://localhost:5000`.

### 8. Open the Frontend

Open `frontend/index.html` in your browser (double-click or use a local server):

```bash
# Using Python's built-in HTTP server
cd frontend
python -m http.server 8080
# Then visit http://localhost:8080
```

## API Reference

| Method | Endpoint      | Description                                      |
|--------|---------------|--------------------------------------------------|
| POST   | `/register`   | Register a user `{ name, email }`                |
| POST   | `/upload`     | Upload a skin image (multipart `file`, `user_id`)|
| POST   | `/predict`    | Predict skin condition from uploaded image       |
| GET    | `/recommend`  | Get product recommendations `?skin_problem=Acne` |

### Example: Full Analysis via cURL

```bash
# 1. Register (optional)
curl -X POST http://localhost:5000/register \
     -H "Content-Type: application/json" \
     -d '{"name": "Alice", "email": "alice@example.com"}'

# 2. Predict skin condition from an image file
curl -X POST http://localhost:5000/predict \
     -F "file=@/path/to/face.jpg"

# 3. Get product recommendations
curl "http://localhost:5000/recommend?skin_problem=Acne"
```

## Testing

Run the automated pipeline tests (no trained model required):

```bash
python scripts/test_pipeline.py
```

The test suite verifies:
- Database tables and product seeding
- User registration (including duplicate email rejection)
- Image upload
- Prediction endpoint (mocked model inference)
- Product recommendation endpoint
- Input validation

## Database Schema

```sql
users          (id, name, email, created_at)
images         (id, user_id, image_path, uploaded_at)
predictions    (id, image_id, predicted_class, confidence, predicted_at)
products       (id, product_name, brand, price, associated_skin_problem)
recommendations(id, image_id, product_id)
```

## Features

- 📷 **Live camera capture** using the browser's MediaDevices API
- 📁 **File upload fallback** for devices without camera access
- 🔬 **CNN model** with 4 convolutional blocks + batch normalisation
- 🛍️ **Product recommendations** filtered by predicted skin condition
- 💰 **Price display in Rs** (Pakistani Rupees)
- 🌙 **Dark mode** with localStorage persistence
- 📱 **Fully responsive** layout for mobile, tablet and desktop
- 🔒 **Parameterised SQL queries** to prevent SQL injection
